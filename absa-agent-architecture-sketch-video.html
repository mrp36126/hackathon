Slide 15: Technical Architecture

Absa Agent orchestrates platforms. It does not replace them.

1. Data Layer

Bring customer signals and outcomes together as live context.

Customer profile Transactions CRM history Digital behaviour Consent Affordability

2. AI Agent / Decision Layer

Understand intent, check rules, rank the next best action, and explain why.

Context builder Intent detector Rules engine Policy guardrails Explanation Learning module

3. API / Execution Layer

Turn the decision into real work through existing Absa services.

API gateway Prequalification CRM task Notification Appointment Analytics

4. Channels

Deliver the action to customers and employees where they already work.

Absa app Online banking CRM Call centre Branch SMS / WhatsApp
The punchline The architecture connects what Absa already has, so journeys can be smarter without replacing core platforms.
Slide 15 shows a practical enterprise architecture for Absa Agent. First, existing data becomes a live customer context. Then the AI Agent decides the next best action with rules, guardrails, and explainability. Next, APIs turn that decision into execution across Absa services. Finally, channels deliver the experience to customers and employees. Outcomes feed back into the data layer, so the next decision gets smarter.
